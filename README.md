### The Skills arc

I designed a [concept product for Claude Skills](https://skills-roan.vercel.app) — persistent instruction sets framed for non-engineers — months before Anthropic shipped the feature natively. The bet: call it a "skill" instead of a "system prompt" and the technical barrier disappears.

Then I built real ones:

- **[code-builder](https://github.com/hbschlac/code-builder)** — 5 parallel drafts in isolated git worktrees, self-scored against a 100-point rubric, winner merged, losers cherry-picked for gaps. Includes a structured debug loop, pre-flight safety checks, and a weekly learning sync that mines post-merge diffs for what the skill got wrong.
- **[mcp-contributor](https://github.com/hbschlac/mcp-contributor)** — 15,000-word skill covering the full MCP governance org (spec, 10 SDKs, SEP lifecycle, contributor ladder). Automated weekly drift detection via GitHub Actions catches when upstream docs change.
- **[Claude Wishlist](https://schlacter.me/claude-ideas)** — a scheduled agent that scrapes r/ClaudeAI 4x/week, categorizes what developers are asking for, and renders a PM dashboard with weekly deltas.
- Plus private skills for interior design, config management, and [session analytics](https://github.com/hbschlac/claude-code-insights-dashboard).
